API Docs

This page describes each of the available API endspoints and what data to expect when calling them

Todo

GET /api/todo/

Returns a list of todo objects

parameters

none

response

        {{
          "[
          {
            id: number,
            title: string,
            description: string
          }
        ]"
        }}
GET /api/todo/:todoId

Returns a specific todo list with its associated tasks

parameters

todoId - integer

response

        {{
          "[
          {
            id: number,
            title: string,
            description: string,
            tasks: [
              {
                id: number,
                taskname: string,
                completed: boolean
              }
            ]
          }
        ]"
        }}
PUT /api/todo/:todoId

Update the title and description of a todo item

parameters

route params
todoId - integer

body 

        {{ "{
          title: string,
          description: string
        }" }}

response

200 ok

POST /api/todo/

Create a new todo item with a specified title and description

parameters

body 

          {{
          "{
            title: string,
            description: string
          }"
        }}

response

200 ok

DELETE /api/todo/:todoId

Deletes a todo with the specified id if the if exists.

parameters

route params
todoId - integer

response

200 Ok

Task

GET /api/task

Fetches a list of all tasks.

parameters

none

response

        {{
          "
          [
            {
              id: number,
              taskname: string,
              completed: boolean,
              todo_id: number
            }
          ]
        "
        }}
GET /api/task/:taskId

Retrieves a single task by a given id, if the id exists

parameters

route params
taskId - integer

response

          {{
          "
            {
              id: number,
              taskname: string,
              completed: boolean,
              todo_id: number
            }
          "
        }}
PUT /api/task/:taskId

updates the taskname and completion status of a todo by a given id

parameters

route params
taskId - integer

body 

        {{
          "
          {
            taskname: string,
            completed: boolean | [0, 1]
          }
        "
        }}

response

200 Ok

PUT /api/task/:taskId/complete

Convenience method for marking a specific task as complete

parameters

route params
todoId - integer

response

200 Ok

POST /api/task

Create a new Task

parameters

body 

        {{
          "
          {
            taskname: string,
            completed: boolean | [0 ,1],
            todo_id: number
          }
          "
        }}

response

200 Ok

DELETE /api/task/:taskId

Delete a todo item with the matching id

parameters

route params
todoId - integer

response

200 Ok
